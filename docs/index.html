    Oversea

    Version of Oversea Application using Ionic 7 & Capacitor 6

    Machine Setup

    In Terminal, install homebrew:

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

    Install node:

    brew install node

    Run the following commands (Note: If you experience permissions issues, use the 'sudo' command):

    npm root
    npm root -g
    npm config set prefix /usr/local
    npm root -g
    npm install -g @ionic/cli //if this doesn't work, run "npm install -g @ionic"
    npm install -g @angular/cli
    npm install -g cordova 
    npm install -g gradle
    npm install -g cocoapods
    npm install -g android studio
    npm i @popperjs/core
    npm i uuid

    Run the following command:

    npm list -g --depth=0

    This should print out a list similar to the following:

    ├── @angular/cli@16.0.2
    ├── android@0.0.8
    ├── capacitor@0.5.5
    ├── cocoapods@0.0.0
    ├── corepack@0.15.1
    ├── gradle@1.2.4
    ├── ionic@5.4.16
    ├── ionic-cli@0.0.0
    ├── n@9.1.0
    ├── npm@6.14.17
    └── studio@0.13.5

    Install XCode: https://developer.apple.com/download/more/

    Install Command Line Tool (CLT): https://developer.apple.com/download/more/

    xcode-select --install

    Install Android Studio and make sure it can open: https://developer.android.com/studio/?gclid=CjwKCAjwnPOEBhA0EiwA609ReaFhODxZ_Xc0efU2IM0jRQRF6LnP8BEAJmsy3EOW-I57Rx7CBuu2HxoCKdIQAvD_BwE&gclsrc=aw.ds

    Install and open VSCode: https://code.visualstudio.com/Download

    Clone the working repository

    git clone https://github.com/PicaPD/OverseaApp.git

    Change the branch from Master to Dev in the bottom left corner of VSCode

    Open new window of VSCode for OverseaApp directory only

    go to the "src/environments" directory and open the "environment.android.ts" file

    follow the instructions in the file, changing the ip in the file under "api_url" and "allowed domains" to your local ip (should be 10.x.x.x)

    your local ip can be found by running "ifconfig" and checking the "en0" section. it will be next to the "inet" label in the indented list

    Android Virtual Device Test Setup

    Open Android Studio and create a new project

    Open the SDK Manager (top right)

    Under the SDK Platforms tab, select "Android 14 (34)"

    Under the SDK Tools tab, select:

    • "Android SDK Build-Tools 35"
    • "Android SDK Platform-Tools 34"

    Click ok to install

    Run the following commands in a BASH terminal in VSCode in the OverseaApp directory;

    brew install jenv
    echo ‘export PATH=”$HOME/.jenv/bin:$PATH” ‘>>~/.zshrc
    echo 'eval "$(jenv init -)" ' >> ~/.zshrc

    RESTART YOUR BASH TERMINAL NOW THEN CONTINUE (still in OverseaApp directory)

    brew install openjdk@17

    If running Apple Silicon:

    jenv add /opt/homebrew/opt/openjdk@17

    If running Intel:

    jenv add /usr/local/opt/openjdk@17/bin/java

    NOTE: If you get a "No such file or directory" error, create directory ~/.jenv/versions if it does not exist, and try running above jenv add command again.

    Run:

    jenv versions

    Output should include:

    17.0
    17.0.11
    openjdk64-17.0.17

    Continue to run:

    jenv global openjdk64-17.0.11
    java -version

    Should print:

    openjdk version "17.0.11" 2024-07-16
    OpenJDK Runtime Environment Homebrew (build 17.0.11+0)
    OpenJDK 64-Bit Server VM Homebrew (build 17.0.11+0, mixed mode, sharing)

    Continue to run:

    /usr/libexec/java_home -V | grep jdk

    Should print (similar):

    17.0.11 (arm64) "Homebrew" - "OpenJDK 17.0.11" /opt/homebrew/Cellar/openjdk@17/17.0.11/libexec/openjdk.jdk/Contents/Home

    Copy this path, and run:

    export JAVA_HOME=the/path/you/copied/before
    export PATH=$JAVA_HOME/bin:$PATH

    Run the following commands:

    npm i
    ionic build -c android
    npx cap sync
    npx cap build android

    Open the Android Studio project you created earlier and open the AVD Manager (top right)

    Under the actions tab (far right) click to install system image

    Once install, open the device to change to your desired device

    Click the green arrow under the actions tab to open the AVD

    Once the home screen has loaded on the AVD, press the green arrow next to the virtual device's name, it will be labeled "run 'app'"

    IOS Virtual Device Test Setup

    Move the download of XCode from downloads to Applications

    Go to Applications in Finder

    View → Show Path Bar

    Control + Click on XCode

    Hold down Option

    Click on “Copy ‘XCode’ as Pathname”

    Paste on sticky note

    Should print:

    /Applications/Xcode.app

    In VSCode Terminal and in OverseaApp directory run:

    sudo xcode-select -s/Applications/Xcode.app/Contents/Developer
    npm i
    ionic build -c android //for simplicity's sake point it to the android environment, this lets it run either android or ios with the same build command
    npx cap sync
    npx cap build ios

    Once Xcode has opened and built the emulator, click the arrow button in the top left next to the "App" title
